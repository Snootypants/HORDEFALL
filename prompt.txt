That is a nasty benchmark because it exposes whether the architecture is actually separated or just looks separated.

Second test: deterministic replay system.

Add deterministic replay support.
Requirements:
- Record player input commands per simulation tick.
- Record seed, version, config hashes, and initial run state.
- Replay the run headlessly and verify final checksum.
- Add a replay viewer in the browser with pause, resume, step-forward, fast-forward, and free camera.
- Add tests for replay determinism.
- If determinism is impossible somewhere, identify the exact nondeterministic system and fix it.

This is a real frontier-level test. Most models will claim “deterministic” while still using Math.random, frame delta, object iteration order, real time, or floating-point chaos everywhere.
